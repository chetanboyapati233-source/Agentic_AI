Jargon → Plain English GLOSSARY

Token
A word-piece. 1 token ≈ 4 chars or ¾ of a word.
Embedding
Text turned into numbers (vector) so similar meanings are close.
Context window
Max input size (Gemini 1.5: up to 1M tokens).
Temperature
0 = deterministic, 1 = creative. Use 0.1 for extraction.
Top-k / top-p
Other ways to control randomness.
Hallucination
Model confidently makes things up. Grounding with RAG reduces it.
Grounding
Forcing the model to answer from sources you provide.
Chunk
A slice of a document (usually 200–1000 tokens).
ANN
Approximate Nearest Neighbor — fast vector search.
SFT
Supervised Fine-Tuning — teach with (input, output) pairs.
Agent
LLM that can pick and call tools in a loop.

5 Prompt Patterns You Need PROMPTS

  • Zero-shot: "Classify as positive/negative: '…'" — simple tasks.
  • Few-shot: Show 2–5 input→output examples, then ask. Best ROI.
  • Chain-of-Thought: "Think step by step." Helps math/reasoning.
  • Structured output: Pass response_schema → guaranteed JSON.
  • Role prompting: "You are a senior SRE. Explain…"
Golden rule: System instruction = rules. User turn = inputs. Keep them separated so prompt injection is harder.

Safety Checklist PROD

  • PII Scrub with Cloud DLP before sending to LLM.
  • Injection Separate system/user; validate outputs; Model Armor.
  • SQL Allow-list SELECT; ban DML/DDL; cap scan bytes.
  • Cost Budget alerts, per-user rate limits, prefer Flash.
  • Obs Log prompt+response hashes, latency, tokens.
  • Eval Golden set in git; LLM-as-judge on every deploy.

Fine-Tuning: Don't Do It (Usually) SFT

Prompting + RAG solves ~90% of problems. Only fine-tune when you need:

  • A consistent style/tone that prompting can't lock in
  • A strict output format with thousands of examples
  • Domain vocabulary the base model doesn't know
Fine-tuning does NOT teach new facts. Those still come from prompts or RAG. Budget: $10–50 per tuning run.

Interview Day Cheat Sheet INTERVIEW

Top 10 questions to nail

  1. Explain RAG like I'm 5
  2. When do you fine-tune vs. use RAG?
  3. How does an embedding work?
  4. Flash vs. Pro — when, why?
  5. How do you evaluate an LLM app?
  6. What's prompt injection; how defend?
  7. Walk through a RAG system design
  8. How do you reduce hallucinations?
  9. Chunking strategies you've used?
  10. How do you control LLM costs?

System design formula

  1. Clarify: users, scale, latency, budget
  2. API: endpoints, request/response shape
  3. Data: sources, ingestion, refresh cadence
  4. Index: chunking, embeddings, vector DB
  5. Retrieval: top-k, rerank, filtering
  6. Generation: model choice, prompting, grounding
  7. Eval: offline golden set + online A/B
  8. Safety: PII, injection, rate limits, budget
  9. Ops: monitoring, logging, on-call, cost
  10. Trade-offs: name 2 you explicitly rejected
